# 使用指南

> 这是给终端用户看的「怎么用 AIreader」。不涉及代码，只讲操作。

---

## 1. 第一次启动

### 1.1 选书库

AIreader 不动你的原始文件 — 它只在自己的设置里记一个「书库根目录」，每次扫描从这里递归找书。

操作：

1. 打开 app。
2. 点右上角「设置」（齿轮图标）→「书库根目录」→「选择文件夹」。
3. 选你放电子书的根目录（比如 `D:\Books`）。
4. 关闭设置面板，回到书架。
5. 点「扫描书库」。

扫描完会显示 `扫描 N 本 · 新增/更新 M · 移除 K`：
- **N**：扫描到的总文件数（已过滤掉不像书的）
- **M**：本次新加的 + 元数据有变化的
- **K**：库里有但磁盘上消失的（被清理掉）

### 1.2 配 AI（可选，但强烈推荐）

不配 AI 也能纯阅读，但失去：问答、自动分类、推荐、汇总笔记、章节配乐、划词查询。

操作：

1. 右上角「设置」→「AI 接口」。
2. 三个字段：

   | 字段 | 填什么 |
   |---|---|
   | base_url | LLM 网关地址，例如 `https://api.deepseek.com`。**不要带 `/v1`**。 |
   | api_key | 你的 API key |
   | chat_model | 模型 ID，例如 `deepseek-chat`、`gpt-4o-mini` |
   | temperature | 可选，默认让模型自己决定 |

3. 点保存。

推荐组合：
- **DeepSeek**（性价比最高）：`https://api.deepseek.com` + `deepseek-chat`
- **OpenAI**：`https://api.openai.com` + `gpt-4o-mini`
- **本地 Ollama / vLLM**：填你自己的网关 URL，确保对外暴露 OpenAI 兼容 API。

## 2. 阅读

### 2.1 打开一本书

书架点任意卡片 → 进入阅读视图。布局：

```
┌─────────────────────────────────────────────────────┐
│  ← 返回书架   字号 - +   主题   AI   标注   音乐    │
├──────┬────────────────────────────────────┬─────────┤
│ 目录 │            正文                    │         │
│ ├ 1  │                                    │ (AI/    │
│ ├ 2  │      ……正文内容……                  │  标注   │
│ └ N  │                                    │  抽屉)  │
└──────┴────────────────────────────────────┴─────────┘
```

### 2.2 翻页 / 跳章

- 鼠标滚轮 / 触控板顺滑滚。
- 滚到本章末尾自动加载下一章拼接，可一路读完整本书。
- 左侧目录点章节直接跳。
- `Esc` 不退出（避免误触），用左上角「返回」。

### 2.3 涂高亮 / 写注释

1. 鼠标选中正文文字。
2. 选区上方弹出小工具栏。
3. 点 5 色之一 → 涂高亮立刻入库。
4. 已涂的高亮再点 → 弹注释编辑框 + 删除按钮。

### 2.4 调字号 / 主题

顶栏「设置」按钮 → Reader 设置面板：

- 字号 14–24（默认 18）
- 行距 1.6–2.4（默认 2.0）
- 列宽 36–56 字符（默认 44，决定正文最大宽度）
- 字体：衬线（默认，Noto Serif SC，更像书） / 无衬线（Inter，更现代）
- 主题：牛皮纸（默认） / 白底 / 深色
- 段落首行缩进开关
- 目录边栏默认是否展开

改完立即生效 + 落库，下次打开同样的偏好。

### 2.5 续读

关掉 app → 重开 → 进同一本书 → 自动回到上次的章节 + 滚动位置。

## 3. AI 问答

阅读视图右上角「AI」按钮 → 弹右侧抽屉。

### 3.1 三种模式

抽屉顶部模式切换：

| 模式 | 适用场景 | 需要先索引 |
|---|---|---|
| **当前章节** | "这章在讲什么"、"刚才那段什么意思" | 否 |
| **整本书** | "整本书的主线"、"作者对 X 的态度" | 是，先「索引本书」 |
| **全书库** | "我看过的书里有没有讨论过 X" | 是，每本要单独索引 |

### 3.2 索引一本书

1. 切到「整本书」模式。
2. 第一次会显示「索引本书」按钮（首次还会下载嵌入模型 ~120 MB，只下一次）。
3. 点按钮 → 进度条显示「N / M 章」。
4. 中等小说几十秒到几分钟。
5. 索引完显示 `本书 · K 片段`，可以开问。

### 3.3 流式回答

- 问完一字一字打出来，不会有 10 秒空白。
- 错误（API key 错、超时、断网）抽屉底部红字显示。

### 3.4 引用跳转

「整本书」/「全书库」回答里会出现`片段 1`、`片段 2`这样的小药丸（带边框）。

- 鼠标悬停 → tooltip 显示对应片段的前 60 字 + 章节号。
- 点击 → 关闭 AI 抽屉，跳到原文章节。

### 3.5 清空 / 历史

- 抽屉右上「清空」按钮：删当前会话所有消息。会话粒度按 `(书, 模式, 章节)` 切：换章后再点开「当前章节」是新对话。
- 不点清空，关掉 app 后再开同一本同一章节 → 历史还在。

### 3.6 划词速问

不想开抽屉？

1. 选中任意一句话 / 一个词。
2. 选区工具栏右侧「✦ 问 AI」按钮。
3. 选区下方弹气泡，30–60 字快速解释 / 翻译 / 文化背景。
4. `Esc` 关掉。

适合：生僻词、典故、引语、跨语种术语。

## 4. 笔记

顶栏「笔记」按钮 → 进 NotesView。

### 4.1 跨书搜索

顶部搜索框输入关键词 → 模糊匹配所有书的所有 highlight（搜 `selected_text` / `note` / 书名 / 作者）。

### 4.2 跳回原文

点任意一条 highlight → 跳到那本书的对应章节，原 highlight 闪烁提示。

### 4.3 导出 Markdown

「导出」按钮 → 生成一份 Markdown：

```md
# 《XXX》/ 作者

> highlight 文字 1
我的注释 1

> highlight 文字 2
…
```

按书分组，便于后期搬到 Obsidian / Notion。

### 4.4 AI 章节汇总

进入一本书 → 顶栏「标注」按钮 → AnnotationsPanel 显示当前书所有 highlight。
- 顶部「✦ AI 汇总」按钮 → 把全部 highlight + note 喂给 LLM。
- 输出 5–7 条要点 + 1–2 条主线（你反复回到的主题）。
- 适合：读完一本书做最终归纳。

## 5. 书架管理

### 5.1 搜索 / 排序 / 筛选

书架顶部：
- 搜索框：模糊匹配标题 + 作者（输入即时过滤）。
- 排序：最近添加（默认）/ 最近阅读 / 标题 A-Z / 作者 A-Z
- 分类 chip：按 11 类分类筛选（需先跑 AI 分类）

搜索词 + 排序选择会记住，重启 app 保持。

### 5.2 AI 自动分类

「AI 分类」按钮 → 把没分类的书每 20 本一批发给 LLM，从 11 个类别里选一个：

`文学小说 · 历史 · 哲学 · 科技 · 经管 · 心理 · 艺术 · 诗歌散文 · 教材工具书 · 传记 · 其他`

进度条显示。完成后所有书都有分类。
- 想重新跑：「AI 分类」按钮长按 / 进阶选项「强制重跑」。

### 5.3 AI 推荐

书架右侧 / 底部「推荐」面板：
- 基于阅读时长 + 分类 + 内嵌相似度算 top-K。
- 每个推荐附 LLM 写的「为什么推荐」一句话。
- 点推荐卡片直接打开那本书。

## 6. 音乐（背景播放）

### 6.1 选音乐库

设置 → 「音乐根目录」→ 选你放音乐的文件夹（含 mp3 / flac / wav / m4a / ogg / aac / **ncm**）。

### 6.2 播放

顶栏「音乐」按钮 → 列出所有曲目。点任一首播放。
- 右下角始终有一个 **mini player** — 切书 / 换页 / 跳目录都不打断。
- NCM（网易云加密格式）首次播放时自动解密，可能稍慢；之后用缓存秒开。

### 6.3 给音乐打 AI 标签

第一次用「章节配乐推荐」之前要先打标：

1. 音乐视图 → 「AI 打标」按钮。
2. 每首曲目根据文件名让 LLM 猜 mood + 描述。
3. 进度条。完成后所有曲目有 mood tags + description + embedding。
4. 加新曲目后再点一次 — 只跑新增 / 文件改过的，已打的跳过。

### 6.4 章节配乐

阅读视图顶栏「音乐」按钮（不是顶栏「音乐」视图，是 reader 内的）→ MusicSuggestPanel：

- 「为本章推荐」按钮 → LLM 给本章打情绪标签 → 在已打标的曲库里语义检索 → 返回 top-K。
- 点哪首播哪首。
- 已经在播的曲不会被打断。

## 7. 隐私 / 离线

- 你的书、笔记、聊天记录**全部在本地**：`%APPDATA%\com.aireader.app\`。
- 嵌入计算 **100% 本地**（fastembed-rs 用 ONNX runtime 跑在你的 CPU 上）。
- 唯一出网：你主动问 AI 时 → prompt 发到你**自己配的** LLM 网关。
- 不上传任何 telemetry，不收集任何使用数据。

想完全断网：拔网线 / 关 Wi-Fi 也能用 — 只是 AI 功能不可用。其他所有功能（阅读 / 标注 / 笔记 / 音乐 / 全文搜索）正常工作。

## 8. 数据备份 / 迁移

所有数据在一个目录里：

```
%APPDATA%\com.aireader.app\
```

复制整个目录 → 新机器同样位置粘贴 → 设置 / 进度 / 标注 / 历史全部迁移。
（书 / 音乐文件本身是按路径引用，需要保证原路径在新机器也存在 — 推荐两台机器用同样的盘符 + 路径结构。）

如果路径变了，最简单：
1. 复制 DB 但保留新机器的 `library_root`。
2. 删除 books 表全部记录（`DELETE FROM books`，CASCADE 会清掉 highlights / chunks / chat 等）。
3. 在新机器重新扫描。
4. 这样会丢标注 / 笔记 / AI 索引，但保留 reader/AI 设置。

更优雅的迁移工具是 roadmap 上的事。

## 9. 常见问题

### Q: 扫描书库扫到 5000 个 README.md 怎么办
AIreader 已经做了启发式过滤（大小 < 5 KB / 文件名黑名单 / 系统目录跳过）。如果还有漏网，提 issue 附上文件路径。

### Q: 某个 PDF 文字是乱码
通常是老 PDF 字体字典损坏，pdf.js 解不了 unicode。建议这本转 EPUB 后再读。AIreader 已经做了 CMaps + standard_fonts + useSystemFonts 三重保险，但有些 PDF 本身就是没救的。

### Q: AI 问答说「未找到相关内容」
你在「整本书」/「全书库」模式下问，但还没索引这本书。切回「当前章节」模式可以问，或者点「索引本书」。

### Q: AI 提示 401 / 404
- 401：API key 错了。
- 404：`base_url` 末尾带了 `/v1` → 去掉；或者模型名 chat_model 不对。
- 其他错误码：去查你的 LLM 提供商文档。

### Q: 索引一本书要多久
取决于书的字数 + 你的电脑性能。中文小说 30 万字大约 30 秒（CPU 推理）。首次还要加上下载 ONNX 模型的时间（~120 MB）。

### Q: 嵌入模型可以换吗
现在 hardcoded 用 BGE-Small-ZH-V1.5。改 `src-tauri/src/ai/embed.rs::EmbeddingModel::BGESmallZHV15` 换成 fastembed 支持的其他模型，重编。如果你的库以英文为主，改成 `BGESmallEN` 检索效果会更好。

### Q: 想分享笔记给朋友
「笔记」视图 → 「导出 Markdown」→ 把生成的 .md 发出去。

### Q: 想完全卸载
1. 删 app 本体（控制面板 / 设置 - 应用）。
2. 删数据目录：`%APPDATA%\com.aireader.app\`。
