# v0.2.2 — NCM 自实现解密器（修网易云音乐不能播）

## 🐞 NCM 音乐不能播

之前用 `ncmdump` crate (0.7 + 0.8 都试了) 解 NCM 文件，`get_data()` 对部分网易云文件返回的**仍然是加密字节**——cache 落盘成 `.mp3` 但实际是密文，HTML5 audio 解码失败但**前端 audio.error 不一定触发**，用户看到的就是"点了不响也没报错"。

### 修法：自实现 NCM 解密器

NCM 算法是公开的（reverse-eng 报告多）：

```
NCM 文件结构 (LE):
  magic "CTENFDAM" (8) + pad (2)
  key_len (u32) + enc_key   ← XOR 0x64 → AES-128-ECB → strip "neteasecloudmusic"
  meta_len (u32) + enc_meta ← XOR 0x63 → strip "163 key(Don't modify):" → base64 → AES-128-ECB
  CRC32 (4) + pad (5) + cover_len (u32) + cover_bytes
  audio_data               ← 用 audio_key 生成 256-byte key_box (RC4 变体), 每字节 XOR
```

按 [CLAUDE.md 原则 1 实践检验](.) — 第三方 crate 解错就自己写。新实现 ~200 行 + 8 个单测（magic byte 6 + key_box 排列性 + AES round-trip）：
- `aes` / `cipher` / `ecb` crate 升为 direct deps
- 解密后用 magic byte 自动检测真实格式（mp3 / flac / wav / m4a / ogg / aac），不信 metadata 声称的 format
- 同时改进前端 `audio.error` 显示：`MEDIA_ERR_DECODE` / `MEDIA_ERR_SRC_NOT_SUPPORTED` 等具体 code 直接展开成中文，方便后续 debug

### 验证

实测一首 `Beyond - 喜欢你(Live).ncm`：
```
NCM decrypt complete (hand-rolled)
  bytes=11408502
  declared_format=mp3
  detected_format=Some("mp3")  ← magic byte 检测成功 ✓
  chosen_format="mp3"
```

cache 文件头从 `db 21 87 41...`（密文）变成 `49 44 33...`（ID3 = MP3）。能正常播。
