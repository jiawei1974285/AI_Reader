# v0.2.1 — 用户反馈修复 + 阅读体验改进

> 7 个 commit 修了 v0.2.0 上用户实际反馈的 6 个问题/改进 + 1 个发现的分页严重 bug。

## 🐞 Bug fix

### 问题 1：MOBI / AZW 日韩书被错判 garbage（盐野七生《文艺复兴的故事》整本不显示）

之前 `decoded_body_is_garbage` 只把 `0x4E00..0x82FF` 算 "可读 CJK"，日语 hiragana / katakana / 常用日韩汉字扩展区全被算成"不可读"。整本日语书 readable_ratio 低于 0.45 阈值 → 被错判 garbage → 返回"MOBI/AZW 压缩格式无法可靠解析"。

修：CJK 范围扩到 `0x9FFF`（覆盖几乎所有日韩中常用汉字）+ 加 Hiragana / Katakana / CJK 标点 / Hangul / Halfwidth-Fullwidth Forms 为 readable。加 2 个回归测试。

### 问题 2：标注跨段落只看到一根竖线

之前 `wrapRange` 在跨段落时 `surroundContents` 抛错 → fallback 走 `extractContents` 把段落分隔（`<br>` / `</p><p>`）一起塞进单个 `<mark>`，渲染出来段间一根竖线。

修：新增 `wrapRangePiecewise`，对 range 跨过的每个 text node 分段 wrap，段落边界保留；多个 mark 共享同一 hl.id，点击反查仍 OK。

### 问题 3：书内检索（Ctrl+F）找到了但不定位

之前用 `window.find()`，在 WebView2 + CSS column 分页模式下浏览器原生 selection 和 reader 的 scrollLeft 没绑定，结果"7/18 但屏幕没动"。

修：DOM-based — TreeWalker 自己 scan 命中 + 包成 `<mark.search-hit>`，当前命中加 `.search-hit-current` 醒目橙色 + `scrollIntoView({block:center, inline:center})`。分页模式下 scrollIntoView 自动调 column 容器的 scrollLeft 到对应列。关闭面板时 unwrap 干净。

### 分页严重 bug：两侧出现"切一半的列"（v0.2.1 新发现）

article 有 `px-10 md:px-16` 横向 padding，导致 column-width 被浏览器缩到 article 实际宽度，而 scroll-snap 仍按 container.clientWidth 步进 → 每页累计偏 16-64px → 多翻几页后两边出现上 / 下页 column 的切片。

修：用一组等式约束 `column-width + column-gap` 严格等于 `container.clientWidth`：
- `.reader-page-paged` 加 `padding-inline: 2rem`（32px）
- 分页模式下 article 取消 `px-10 md:px-16`
- `column-gap: 4rem` (= 2 × container padding-inline)
- `getPagedStep` 返回 `root.clientWidth`（不再 +gap）

## ✨ 改进

### 改进 1：实体抽取一键导出 Markdown（整本汇总）

`ChapterEntitiesPanel` 加「↓ 导出全书实体 .md (N 章)」按钮，按 spine_index 排序遍历已抽过的章节，每章一个 `##` section，人物 / 地点分组列出 (`- **名字** — 简介`)。顶部带书名 / 作者 / 章节数 / 实体总数 / 导出时间。

### 改进 2：分页翻页 3D rotateY 动画

CSS `perspective: 1800px` + `@keyframes` 320ms rotateY ±14deg + opacity dim。`goToNextPage` / `goToPrevPage` 触发 `flipDir` state，360ms 后自清。不引入第三方库。比直接 jump 有翻动反馈，但仍不是完整 page-curl（page-curl 需要 canvas/SVG filter）。

### 改进 3：翻页按钮挪到正文下方

底部进度条 strip 旁加 `[← 上一页]` / `[下一页 →]`（仅分页模式）。header 顶部按钮保留作 fallback，手 / 鼠标不用跨屏到顶。
